Introduction to Lobs (Layout Objects)

Lobs alias Layout Objects is area in Fenfire project which is work in progress. We aim to have it ready for use shortly after this lecture to help you to start your programming project.

The need for yet another layouting system was obivious since:

  • swing has not free implementation.
  • awt is not enough.
  • we were not interested in java2d which many libraries depend on.
  • we had already choosen to use Java as the programming language.
  • none of the above do the proper thing we wanted - system good looking as latex output, conservative PUI toolkit with animation support, more flexibility to make changes than vobs provide, and small memory print.
  • libvob and thus visual objects are a powerful system with its coordinate systems and keys but it needs too much care to handle and for example event handling is not easy - as prooved with FenPDF prototype.

Therefore Benja started to design and construct layouting system which used the power of libvob's visual objects but formalized them more easier and object oriented way.

Those who are too keen to wait to see it in use can check an example of boss buzzle (and if you are interested to see how it differs from a one that uses too powerful visual objects you can see this example written in jython).

The basic idea with Lobs is KISS - do one thing and do it well. This idea is from InterViews/Fresco which has been the main inspiration source for Lobs. These "do one thing well" objects can be combained to create components, e.g., button consist of frame, click controller and content lob which may be a label containing text.
