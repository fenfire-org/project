Introduction to Xanalogical Media

Xanalogical Content

Theodor (Ted) Holm Nelson had an idea in the year 1960 of xanalogical media which can be a huge revolution in computing in the future. What does this technology mean? Let's imagine that every key stroke in computer is identified, and thus when Pekka presses A-letter it's not the same A-letter that Jukka presses in his computer. At the beginning this may sound impossible but it isn't - we just need to forget ASCII for the moment.

Xanalogical media can be implemented as follows. Instead of storing a letter, for example A-letter (0x41), a identity is constructed which refers to pressed A-letter. To programmer this can be expressed as reference, pointer and such. After constructing the identity, the real content shall be stored to refereed place.

Xanalogical media which in this example is text, gives significant advantage compared to common text. Every citation of the text is a real citation and original media can be easily seen as well as the author. For example if you cut text from Keskisuomalainen from the year 1983 on January sencond Sunday, the user interface can show not only the citated tex but also the whole news paper.

Xanalogic media is not only about text or invidual key strokes even if xanalogic text is still the first idea which pops out from the idea. For example in Fenfire we use to show (scientific) articles with images from each page. Other media might be sound and video. It's noticed that even if you can speak of many medias, there's only few which can be realised.

In the following example there is xanalogic text used in the Fenfire system. In practise xanalogic media is orthogonally expressed in RDF graph by putting an XML information in typed literal. 


# An example of use of xanalogical text with TIML implementation.


import org.nongnu.alph as alph

textMerkki = alph.TString.newRICC("merkki")
textEsi = alph.TString.newRICC("Esi")


text = textEsi.plus(textMerkki)
print text
print text.toXML()

text = textMerkki.plus(textEsi)
print text
print text.toXML()

Xanalogic Link

Xanalogic link is the original idea behind current web systems. As it turns out web uses one way links where xanalogic link is two way. In short xanalogic link is two points and between those is a line segment where points are references to xanalogic media.

If we imitate the example above we could say: 

textMerkki = alph.TString.newRICC("merkki")
textEsi = alph.TString.newRICC("Esi")

xuLink = createXuLink(textMerkki, textEsi)
